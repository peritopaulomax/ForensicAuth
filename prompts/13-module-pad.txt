# 13-module-pad.txt - Modulo de Deteccao de Ataques de Apresentacao (PAD)

## Objetivo

Implementar a tecnica forense `presentation_attack_detection` no ForensicAuth, integrando o modelo open source Silent-Face-Anti-Spoofing (MiniFASNetV2 + RetinaFace) como um novo plugin na aba "Imagens", com dashboard proprio, execucao em fila GPU/CPU, cadeia de custodia e testes E2E fim a fim.

## SETUP GUIADO (ambiente da fase)

Antes de codar, conduza o ambiente:

1) Verifique se o ambiente conda `forensicauth` esta ativo:
   ```bash
   conda activate forensicauth
   ```

2) Verifique se PyTorch, OpenCV e dependencias do Silent-Face-Anti-Spoofing estao instalados:
   ```bash
   python -c "import torch; print(torch.__version__)"
   python -c "import cv2; print(cv2.__version__)"
   ```

3) Verifique/instale dependencias do modelo:
   ```bash
   pip install torchvision
   ```

4) Verifique pesos do modelo:
   ```bash
   ls models/pad/anti_spoof_models 2>/dev/null || echo "FALTANDO"
   ls models/pad/detection_model 2>/dev/null || echo "FALTANDO"
   ```
   Se faltarem, baixe do repositorio original:
   ```bash
   git clone https://github.com/minivision-ai/Silent-Face-Anti-Spoofing.git /tmp/silent-face
   # Copie resources/anti_spoof_models e resources/detection_model para models/pad/
   ```

5) Verifique licenca do modelo (CC BY-NC 4.0) e documente restricoes em docs/public/LICENCAS-PAD.md.

6) Valide GPU disponivel:
   ```bash
   python -c "import torch; print(torch.cuda.is_available())"
   ```

### 2. Backend - Adapter

Em src/backend/core/plugins/presentation_attack_detection_adapter.py:
- Herde de ForensicPlugin
- name = "presentation_attack_detection"
- supported_types = ["imagem"]
- validate_parameters(): aceite "threshold" opcional (padrao 0.5)
- analyze():
  - Carregue imagem via PIL/OpenCV
  - Use RetinaFace (vendored) para detectar faces
  - Selecione a face principal (maior bbox)
  - Use MiniFASNetV2 para classificar
  - Retorne dict padronizado:
    ```python
    {
      "success": True,
      "adapter": "presentation_attack_detection",
      "label": "real" | "fake",
      "score": float,
      "bbox": {"x": int, "y": int, "w": int, "h": int},
      "inference_device": "cuda" | "cpu",
      "label_image_path": str,  # imagem com bbox desenhado
      "result_json_path": str
    }
    ```
- Em caso de erro (sem face, falha de modelo):
  ```python
  {"success": False, "adapter": "...", "error": "NO_FACE_DETECTED", "message": "..."}
  ```
- Use run_with_device_fallback para GPU/CPU
- Libere VRAM apos uso
- NAO altere o algoritmo interno do Silent-Face-Anti-Spoofing (Regra Maxima 8)
